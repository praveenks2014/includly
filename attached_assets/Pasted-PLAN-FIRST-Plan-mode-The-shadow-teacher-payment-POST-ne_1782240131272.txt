PLAN FIRST — Plan mode. The shadow-teacher payment POST never reaches
the API server — server logs confirm zero occurrences of
POST /api/shadow-teacher/request. The HTML "<!DOCTYPE" error is the
Replit proxy intercepting an unauthenticated request before it reaches
port 8080.

Root cause: ClerkAuthBridge sets the token getter in a useEffect.
During Clerk session refresh or component remount, the cleanup
temporarily sets the getter to null. Any request fired during this
window goes out without a Bearer token → Replit proxy returns an HTML
sign-in redirect → frontend gets "Unexpected token '<'" parsing it as
JSON.

This is a SYSTEMIC issue — it affects any fetchWithAuth call fired
during a Clerk token refresh. The payment button is just where it's
most visible because payment is high-stakes. The four 401s on
POST /api/engagements/3/lifecycle at 18:24-18:34 are the same issue
reaching the server (that route is on a different path the proxy
doesn't intercept).
