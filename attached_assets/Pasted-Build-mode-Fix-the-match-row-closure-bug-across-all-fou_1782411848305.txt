match-row update sets status = "committed" (which the row already is)

update is always silently skipped. The match row never moves to a
genuinely terminal status, so the frontend reads the stale committed
state and shows the post-commit UI instead of the re-engagement flow.

buyout):

  .set({ status: "committed", updatedAt: new Date() })

    eq(shadowTeacherMatchesTable.id, engRow.matchRequestId),
    notInArray(shadowTeacherMatchesTable.status,
      ["cancelled", "refunded", "committed"]),  // ← skips committed rows
